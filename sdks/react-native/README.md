# @openhorizon-labs/sapient-react-native

**On-device SAPIENT inference for React Native** — the engine runs inside
your app (UniFFI → JSI TurboModule over
[`crates/sapient-ffi`](../../crates/sapient-ffi)), GPU included (wgpu:
Metal on iOS, Vulkan on Android; `auto` probes the adapter and falls back to
CPU). Generated by
[uniffi-bindgen-react-native](https://github.com/jhugman/uniffi-bindgen-react-native).

Requires a **development build** (`npx expo prebuild` / bare RN) — Expo Go
cannot load native code. New Architecture (TurboModules) required — default
since Expo SDK 52 / RN 0.76. Hermes only.

## Use with the TypeScript SDK (recommended)

```ts
import { SapientClient } from '@openhorizon-labs/sapient';
import { NativeTransport } from '@openhorizon-labs/sapient-react-native';

const client = new SapientClient({
  transport: new NativeTransport({ cacheDir: appCachesDir }),
});
// identical API to the serve-backed transport:
for await (const tok of client.chatStream(
  [{ role: 'user', content: 'Hi!' }], 'smollm2-135m-q4'))
  console.log(tok);
```

The raw generated bindings are also exported (`loadSession`, `LlmSession`,
`chatMessagesStream`, `setThermalLevel`, `setCacheDir`, `listModels`, …).

## Building (monorepo)

```bash
npm install
IPHONEOS_DEPLOYMENT_TARGET=14.0 npm run ubrn:ios   # cargo + xcframework + codegen
npm run ubrn:android                                # NDK .so + codegen
```

`ubrn:*` regenerates `src/generated`, `cpp/generated`, the podspec, and the
native glue from the Rust crate, then applies `ubrn:postgen` (ubrn 0.29.3
emits `async public` — invalid TS — which the script reorders).

## Invariants (read before touching)

- **ubrn ↔ uniffi lockstep**: this package pins `uniffi-bindgen-react-native@0.29.3-1`;
  the crate pins `uniffi = "=0.29.3"`. Bump BOTH together (0.31 line next —
  brings the Android 16KB-page-alignment fix).
- `src/index.tsx` is `noOverwrite`-protected in `ubrn.config.yaml`: it carries
  the `NativeTransport` re-export on top of ubrn's installer glue.
- Consuming apps must Metro-block-list this package's own
  `node_modules/{react-native,react}` — see
  [`examples/react-native-chat/metro.config.js`](../../examples/react-native-chat/metro.config.js).
- The FFI calls are the **async** exports — the blocking API would freeze
  Hermes (and `chat_stream` would deadlock). `chatMessagesStream` is
  stateless (caller owns history, like `sapient serve`); the engine's prefix
  cache keeps re-sent history cheap.

including §5, the safe-testing ladder for personal hardware.
